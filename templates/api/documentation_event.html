Получение событий

Событие можно получить как по id: {{ event_by_id_example }},
так и по его name: {{ event_by_name_example }}

Пример ответа:
{
    "type": "event",
    "id": 29,
    "name": "uspenskiy_post",
    "tags": [
        "post"
    ],
    "text": {
        "content": "content of event",
        "annonce": "annonce of event",
        "title": "title of event"
    },
    "image": "Ph4LnbLHNfs_3.jpg",
    "icons": [
        {
            "text": {
                "content": "content",
                "annonce": "annonce",
                "title": "title"
            },
            "image": "4J9TcNQjo_M.jpg",
            "type": "icon",
            "id": 377
        },
        {
            "text": {
                "content": "content",
                "annonce": "annonce",
                "title": "title"
            },
            "image": "fe70858c-420d-49c4-a955-97e9bf2cc10f.jpg",
            "type": "icon",
            "id": 380
        },
    ],
    "articles": [
        {
            "image": "",
            "type": "article",
            "id": 218
            "name": "post_dlya_tela"
        },
        {
            "image": "",
            "type": "article",
            "id": 225,
            "name": "vidy_postnyh_dnyay"
        },
    ]
}
Cледует обратить внимание!
  1. Cекция "статьи" сокращенная - приводится лишь id, name, type, image
  2. Cекция "иконы" полная - приводится в том числе тексты
  3. Секции tags,articles,icons могут отсутствовать
  4. порядок полей ответа может быть иным

События можно получать по тегам: {{ event_by_tag_example }}. При этом список секций будет сокращенных 

Пример ответа:
[
    {
        "image": "image11.jpg",
        "type": "event",
        "id": 11,
        "name": "event11",
        "tags": [
            "tag1",
            "tag2"
        ]
    },
    {
        "image": "image12.jpg",
        "type": "event",
        "id": 12,
        "name": "event12",
        "tags": [
            "tag1",
            "tag3"
        ]
    }
]
